# 嵌入式学习路规划

> 项目：AI 猫（ESP32-S3 毛绒玩具）
> 日期：2026-06-21

## 背景

项目已打通语音对话链路（WebSocket + STT + LLM + TTS），但显示屏点不亮，后续需外接摄像头和舵机。⽤户为软件背景，嵌入式近乎零基础。

**关键认知：** 项目已完成 1870+ 行固件代码（C++17），含 I2S 音频、Opus 编解码、WebSocket、PSRAM 管理、WiFi 等。真实能力远超自评——真正缺的是**硬件调试方法论**和**特定外设知识**，而非计算机或编程基础。

## 核心判断

**不建议「从头学嵌入式」。** 原因是：

1. 软件功底（C++、网络、系统设计）已帮完成最复杂的部分。缺的知识可用 **20-30 小时** 补上，而非 6-12 个月
2. 项目驱动的「即时学习」（Just-in-Time Learning）效率最高——遇到的每个问题都是最好的教材
3. 显示屏不亮是**常见初学绊脚石**——资深工程师也常在 SPI 显示屏上翻车

**推荐方案：混合式学习**——花 4-6 周打硬件基础，同时用项目作实验场，每学一个概念马上在猫项目中应用。

## 学习路线图（6 周计划）

### 第 1 周：硬件调试基本功

**目标：** 能看懂原理图，会用基础工具排查问题

- **必买工具：** USB 逻辑分析仪（Saleae 兼容版，淘宝 ~¥30-50）+ 万用表
- **学习内容：**
  1. 下载立创实战派的**原理图 PDF**（在嘉立创开源硬件平台找），把 AI 猫用到的每个引脚标注出来
  2. 学看原理图：电源轨（3.3V/5V）、信号流向、上拉电阻的作用
  3. 用万用表测：显示屏 VCC 有没有电？背光引脚电压多少？
  4. 练习：把 `firmware/main/ai_cat_board.h` 的引脚定义和原理图一一对照

**与项目结合：** 打开原理图，找到 ST7789 显示屏部分的电路，确认 BL（背光）、DC、RST 的实际连接方式

### 第 2 周：SPI + 显示屏驱动

**目标：** 理解 SPI 协议，让显示屏亮起来

- **学习内容：**
  1. SPI 通信原理：SCLK、MOSI、MISO、CS、DC 各起什么作用
  2. SPI mode 0/1/2/3 的区别（显示配置了 mode 2）
  3. ST7789 初始化序列：什么是 sleep out、display on、color mode
  4. RGB565 色彩格式

- **关键调试方法：**
  1. 用逻辑分析仪抓 SPI 总线：查看 ESP32 是否在发送数据
  2. 检查显示屏上电时序：VCC 要先于 SPI 信号
  3. 注意背光（BL）极性：代码中 BL=42、反相（0=亮, 1=灭），确认与硬件一致

- **参考：** xiaozhi-esp32 项目中 `lichuang_dev_board.cc` 的显示屏初始化流程，注意它调用了 `pca9557_->SetOutputState(0, 0)` 供电

### 第 3 周：I2C + PCA9557 + PCA9685 舵机

**目标：** 理解 I2C，让舵机动起来

- **学习内容：**
  1. I2C 协议：地址、读写、多设备共享总线
  2. PCA9557（GPIO 扩展器）：如何用 I2C 控制 GPIO
  3. PCA9685（PWM 驱动）：如何产生 16 路 PWM 控制舵机
  4. 舵机控制原理：20ms 周期、1-2ms 脉冲、角度映射

- **调试方法：**
  1. 用逻辑分析仪抓 I2C 总线：确认 ESP32 在与 PCA9557/PCA9685 通信
  2. i2c_scanner 工具（ESP-IDF 示例）扫描总线上有哪些设备

### 第 4 周：调试显示屏问题

**目标：** 综合运用前 3 周知识，彻底解决显示不亮

- **系统化排查流程：**
  1. **电源：** 万用表测显示屏 VCC/GND → 有 3.3V 吗？
  2. **背光：** 测 BL 引脚 → GPIO42 输出低电平时，BL 是 0V 还是 3.3V？
  3. **SPI 信号：** 逻辑分析仪抓 SPI 总线 → ESP32 是否发出了初始化命令？
  4. **PCA9557 供电：** 确认 `pca9557_->SetOutputState(0, 0)` 是否执行（参考项目要求这个才能给显示屏供电）
  5. **复位时序：** 部分屏幕需要 RST 拉低再拉高，RST=-1（NC）时如屏幕无内部上拉需处理

- **代码修改点：**
  - 取消注释 `InitializeDisplay()` 调用
  - 在 `EyeDisplay::Initialize()` 中添加 `pca9557_->SetOutputState(0, 0)`（或由 `AiCatBoard` 在调用前设置）
  - 避免使用 `ESP_ERROR_CHECK`（panic），改为检查返回值并打印日志
  - 测试背光是否反相（尝试 `gpio_set_level` 为 1 和 0 两种状态）

### 第 5 周：摄像头接入

**目标：** 添加摄像头（OV2640 或其他接口兼容型号）

- **学习内容：**
  1. 摄像头接口类型：DVP（并行）/ SPI / USB
  2. ESP32-S3 的 Camera 驱动（`esp_camera` 组件）
  3. 帧缓冲管理、JPEG 编码
  4. 与现有音频链路的带宽协调

- **调试方法：**
  1. 先跑通 ESP-IDF 的 camera 示例
  2. 确认引脚分配不冲突（SPI 摄像头与显示 SPI 是否冲突）
  3. 测试帧率和图像质量

### 第 6 周：系统集成 + 打磨

**目标：** 所有模块联动，系统稳定运行

- **内容：**
  1. 舵机 + 显示屏 + 触摸 + 音频 多任务协调
  2. 电池管理（ADC 采样、低电量告警）
  3. 离线模式（WiFi 断开时的本地猫行为）
  4. 电源优化（休眠、唤醒）

## 一个重要的认知

`firmware/main/main.cc` 中已完成 PSRAM 分配、流缓冲管理、音频编解码任务调度——这些很多自认为「会嵌入式」的人也不一定能写好。能力没问题，缺的只是**针对硬件调试的流程性知识**。

把显示屏点亮这个「小目标」当作嵌入式的「成人礼」——一旦搞定，就会建立「硬件也没那么可怕」的信心，后续摄像头和舵机只是换一个外设重复同样的调试流程。

## 即时行动建议

1. **买逻辑分析仪**（最重要工具，等快递期间可学 SPI 理论）
2. 下载立创实战派原理图，与引脚定义做对照
3. 阅读 `ai_cat_board.cc` 中的 `InitializeDisplay()` 方法，加入 `pca9557_->SetOutputState(0, 0)` 供电
4. 取消 `InitializeDisplay()` 的注释，改用 if-err-log-return 模式而非 `ESP_ERROR_CHECK`
5. 用万用表量显示屏 VCC 和 BL 引脚，确认供电和背光状态

## 验证方法

每学完一个知识块，用以下方式验证掌握程度：

1. **SPI：** 能口述「为什么显示屏配置了 mode 2？」「pclk_hz=80MHz 意味着什么？」
2. **I2C：** 能解释「为什么 PCA9557 和 PCA9685 可以挂在同一条 I2C 总线上？」
3. **逻辑分析仪：** 抓取一次 SPI 通信，能分辨出 DC 线切换和数据传输的阶段
4. **显示屏点亮：** 最终验证——屏幕上显示出猫眼睛
5. **舵机转动：** PCA9685 输出 PWM，舵机按指令转到指定角度
6. **摄像头出图：** 服务器收到 ESP32 传来的 JPEG 图像
