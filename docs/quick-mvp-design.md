# AI 猫 · 快速落地方案

> 目标：1 周内跑通完整电子链路，让一只毛绒猫"活过来"
>
> 策略：现成毛绒玩具 + ESP32 电子组 + 3090 AI 服务器

---

## 1. 方案概述

```
┌─────────────────────────────────────────┐
│              毛绒猫玩具                    │
│  ┌─头部────────────────────────────────┐ │
│  │  左耳舵机 · 右耳舵机 (SG90×2)        │ │
│  │  扬声器 (喉咙位)                      │ │
│  ├─身体────────────────────────────────┤ │
│  │  ESP32-S3 主板                       │ │
│  │  MEMS 麦克风 (INMP441)               │ │
│  │  触摸传感器 ×3 (头顶/背/肚皮)         │ │
│  │  震动马达 (呼噜模拟)                  │ │
│  │  电池 18650×2 + 充电模块             │ │
│  │  腹部拉链 → 打开即检修                │ │
│  └────────────────────────────────────┘ │
│         │ WiFi / WebSocket              │
│         ▼                               │
│  ┌─────────────────────────┐            │
│  │   3090 服务器            │            │
│  │   LLM + 猫人格 + 情绪    │            │
│  └─────────────────────────┘            │
└─────────────────────────────────────────┘
```

**核心定位**：这不是最终产品，是一个**链路验证机**。验证：
- ESP32 ↔ 3090 的 WebSocket 通信是否稳定
- 触摸→情绪→表达 的交互闭环是否自然
- 麦克风拾音 + 扬声器发声在实际毛绒环境中的效果
- 电池续航是否够用（目标 8 小时+）
- 舵机在毛绒包裹下的噪音和扭矩表现

验证完成后，电子部分直接移植到长期方案的壳体中。

## 4. 改造步骤

### Step 1：选玩具（1-3 天）
- 淘宝下单猫形毛绒玩具
- 到手后检查：肚子是否有空间、能否塞入 ESP32+电池

### Step 2：开检修口（1 晚）
- 如果原玩具有腹部拉链 → 直接用
- 如果没有 → 在腹部剪开 15cm 口子，手缝隐形拉链
- 拉链打开后，掏出部分填充棉，为电子元件腾空间

### Step 3：焊板子（1-2 晚）

```
ESP32-S3 接线：

I2C-0 (SDA=GPIO41, SCL=GPIO40):
  └── PCA9685 → SG90 左耳 + SG90 右耳

I2S (BCK=GPIO15, WS=GPIO16, DIN=GPIO17):
  └── MAX98357 → 3W 扬声器

I2S (BCK=GPIO4, WS=GPIO5, DIN=GPIO6):
  └── INMP441 MEMS 麦克风

GPIO 触摸（电容触摸）:
  ├── GPIO1 → 头顶触摸铜箔
  ├── GPIO2 → 背部触摸铜箔
  └── GPIO3 → 肚皮触摸铜箔

GPIO 输出:
  ├── GPIO11 → 震动马达 PWM
  └── GPIO12 → 状态 LED

供电链路：
  18650×2 并联 → TP4056 → MT3608(5V) → 舵机
                       └── AMS1117(3.3V) → ESP32 + 传感器
```

### Step 4：塞进去（1 晚）

- 电子组放入玩具内部
- **触摸铜箔**贴在毛绒皮内侧（头顶/背/肚皮），用魔术贴固定
- **扬声器**对准喉咙区域
- **麦克风**固定在头内部，朝向嘴巴/鼻子区域，周围填充薄海绵隔振
- **震动马达**固定在身体侧壁，紧贴毛绒皮内侧传导震感
- **耳朵舵机**装在头部两侧，用热熔胶+扎带固定于玩具内部
- **电池**放在肚子最深处，靠近拉链方便充电
- 线缆整理，用扎带绑好，避免缠绕舵机

### Step 5：固件烧录（1-2 晚）

固件核心模块：
- WiFi 连接 + WebSocket 客户端
- 7 路触摸电容值采集（10Hz）
- 麦克风音频流采集（16kHz, 16bit, mono）
- 舵机角度控制（PCA9685 PWM）
- 震动马达 PWM 调压
- 扬声器音频播放（接收服务器下发的音频数据）
- 心跳包 + 离线模式（断连时进入"睡觉"状态）

### Step 6：服务器端（2-3 晚）

```
WebSocket 服务（Python/Node.js）
  ├── 接收 ESP32 上行数据（触摸、音频、电量）
  ├── STT 语音识别（Whisper/FunASR）
  ├── LLM 猫人格推理（Qwen/Llama 等本地模型）
  ├── 情绪状态机（content/sleepy/playful/annoyed/scared）
  ├── TTS 语音合成 → 下发音频到 ESP32
  └── 动作指令生成（耳朵角度、震动强度）→ 下发 ESP32
```

### Step 7：联调（1-2 晚）

- 摸猫不同部位 → 服务器识别 → 不同的呼噜强度回应
- 对猫说话 → STT → LLM 回应 → TTS 喵叫/语音
- 情绪状态切换测试（一直不理猫 → 进入 sleepy → 呼噜减弱）
- WiFi 断连/重连测试
- 续航实测

## 5. 交互闭环

| 输入 | 处理 | 输出 |
|------|------|------|
| 摸头顶 | 服务器判断"被摸头" + 情绪融合 | 耳朵趴平 + 强呼噜 |
| 摸肚皮 | 服务器判断"被摸肚皮" | 耳朵微动 + 轻呼噜（猫肚皮敏感） |
| 长时间无人摸 | 情绪衰减 → sleepy | 耳朵趴平 + 弱呼噜（模拟睡觉） |
| 说话"过来" | STT → LLM 理解意图 | 耳朵立起 + 喵叫回应 |
| 突然大声 | 麦克风检测到高音量 | 耳朵警觉 + 停止呼噜 |

## 7. 快速方案的价值

- **验证电子链路是否可行**：ESP32 ↔ 3090 的 WebSocket 实时通信是整个 AI 猫的技术基石
- **积累可复用的代码资产**：固件和服务器端代码直接迁移到长期方案
- **暴露问题**：空间布局、散热、触摸灵敏度、电池续航——在便宜的原型上发现比在贵的壳上发现好
- **获得信心**：一只会呼噜、会动耳朵、会回应的猫，即使长得不像你家猫，也足够让你相信这条路走得通
